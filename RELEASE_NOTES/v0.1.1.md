# Moonlight Android SBS Test Release v0.1.1

This is a test release created specifically to validate the corrected release publishing workflow.

## Purpose

The goal of this release is to confirm that APK files appear under the GitHub Release assets section together with GitHub's automatic source code ZIP and TAR archives.

## What this release tests

- Release branch flow using `release/v0.1.1`.
- `VERSION` file validation using `v0.1.1`.
- Release notes loading from `RELEASE_NOTES/v0.1.1.md`.
- Version tag creation from the release workflow.
- Android APK build inside the same release publishing workflow.
- APK collection from Gradle output directories.
- GitHub Release creation.
- APK upload as a release asset.

## Expected result

After the release workflow completes, the `v0.1.1` GitHub Release should show:

- One or more `.apk` files under release assets.
- `Source code (zip)`.
- `Source code (tar.gz)`.

## Important workflow fix included before this release

The release workflow was changed so it no longer depends on a second tag-triggered workflow to upload APK files. Instead, the release PR merge workflow now performs the full publish sequence in one workflow:

1. Validate release branch and version.
2. Build the APK.
3. Collect APK files.
4. Create and push the version tag.
5. Create the GitHub Release.
6. Upload APK files directly to the release.

This avoids the issue where a tag created by GitHub Actions using `GITHUB_TOKEN` may not trigger another workflow.

## Build type

This test release builds the non-root debug APK using:

```bash
./gradlew --no-daemon clean assembleNonRootDebug
```

## Validation checklist

1. Open the `v0.1.1` release page.
2. Confirm at least one APK is visible under release assets.
3. Confirm source ZIP and TAR files are also visible.
4. Download/install the APK on a test Android device.
5. Confirm normal launch behavior.
6. Confirm SBS VR setting is still available for testing.

## Known notes

- This is a workflow validation release, not a final production release.
- The APK may be unsigned/debug depending on the current Gradle variant.
- SBS rendering should still be treated as test functionality.
