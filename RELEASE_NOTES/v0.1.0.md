# Moonlight Android SBS Test Release v0.1.0

This is the first versioned test release for the custom Moonlight Android fork. It introduces the first implementation of an optional side-by-side phone-VR rendering path and establishes the version-based release workflow that future builds should follow.

## Release purpose

This release is intended for mobile-device validation of the new SBS rendering path. It should be treated as an early test build rather than a final production-ready VR implementation.

## Highlights

### Optional SBS VR mode

A new optional SBS VR mode has been added. When enabled, streams are launched through a dedicated SBS activity instead of the normal game activity.

The SBS mode renders the normal Moonlight video stream into an OpenGL external texture, then displays that texture as a duplicated left/right side-by-side image suitable for phone-VR headset testing.

### Mono side-by-side rendering

This release does not implement true stereoscopic rendering. It duplicates the same Moonlight stream into both eyes, matching the behavior of the older SBS fork.

Expected behavior:

- Left half: full Moonlight stream frame
- Right half: same full Moonlight stream frame
- Split point: center of the device screen
- Optional lens-style warp/distortion applied by shader

### SBS shader fixes

The SBS shader was updated so the left/right screen split is based on raw output screen coordinates. The Android decoder texture transform is now applied only when sampling the decoded video texture.

This avoids incorrect SBS splitting caused by decoder crop, flip, or scale transforms on modern Android devices.

### Renderer lifecycle fix

The OpenGL render thread initialization was corrected so the renderer thread starts only after subclass state is initialized. This reduces risk of rendering startup issues caused by the base constructor starting the thread too early.

### Full-screen SBS viewport

The SBS renderer now uses the full output surface for the viewport. This keeps the left/right split aligned to the actual screen instead of applying video-aspect viewport cropping before the SBS split.

## Settings added

The following settings were added under Streaming Settings:

- Enable SBS VR mode
- SBS VR size
- SBS VR distortion
- SBS VR wrap sides
- SBS VR single view debug

## Known limitations

- No head-tracking support yet.
- No true stereo source rendering yet.
- HDR is disabled for SBS mode for now.
- SBS mode is intended for controller-based testing first.
- Touch/mouse behavior may need additional tuning for the SBS activity.
- This is an early test release and may still need device-specific rendering adjustments.

## Recommended test checklist

1. Install the APK on an Android phone.
2. Open Streaming Settings.
3. Enable SBS VR mode.
4. Start a stream from a paired host.
5. Confirm the stream appears as two side-by-side images.
6. Confirm the split is centered on the screen.
7. Test SBS VR size and distortion settings.
8. Disable SBS VR mode and confirm normal Moonlight streaming still works.
9. Test at least one controller input path during streaming.
10. Report whether the SBS image is centered, stretched, cropped, flipped, or otherwise distorted.

## Release pipeline changes

This release also introduces the repository's version-based release process:

- Development should happen on feature, bugfix, hotfix, chore, or release branches.
- Direct pushes to master should be avoided.
- Changes should enter master through pull requests.
- Release branches should use the format release/vX.Y.Z.
- VERSION must match the release branch name.
- Merging a release branch into master creates the matching version tag.
- APK builds and GitHub Releases are triggered by version tags only.

## Artifact

The APK is attached to this GitHub Release after the tag-based build workflow completes.
